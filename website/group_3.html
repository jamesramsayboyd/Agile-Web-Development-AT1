Authentication and web security:
Before sensitive data is transmitted over the web the identity of the user submitting or accessing it should be verified. It is necessary to perform a process of authentication to make sure the user is who they say they are, which is often accomplished by questioning the user about something they should know (such as a security question) or something they should have (e.g. a username and password, or a physical token).
Hypertext transfer protocol (HTTP):
The protocol used to transfer data over the World Wide Web. It uses a server-client model, where the client (e.g. a personal computer or mobile device) running an internet browser sends a request to a server (the web host running a website on server software such as Apache), which will respond with an HTTP status code. For example, an HTTP code of 200 indicates a successful request, and will result in the server sending the data to display the desired webpage in the browser.
Session management:
A session is defined as a series of HTTP transactions initiated by a single user, that typically starts when the user authenticates their identity (through the use of a password, for example). Session management is the process of handling these multiple requests, which may be sensitive in nature, while maintaining security and continuity with the same user.
Stateless programming:
A program is said to be ‘stateless’ when it does not take information about the last session (data such as user configuration settings or transaction information) into the next session. The HTTP protocol used for websites is an example of stateless programming, where no information is maintained from session to session, and countermeasures such as cookies must be used to maintain a state between web pages.
