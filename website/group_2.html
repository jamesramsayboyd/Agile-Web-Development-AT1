What are programming controls and design structures?
The three basic types of programming controls are Sequence (the order in which statements are read by the compiler), Selection (which statements are compiled, governed by set conditions) and Iteration (the repetition of a statement a set number of times). Programming should be structured in such a way as to improve the clarity of code and reduce the development time of a project by, for example, using Iteration constructs (while and for loops) or Selection constructs (if/else) to keep code simple and concise and minimise repetition.
What are website testing procedures?
  • Functional Testing: A type of black-box testing that focuses solely on the user experience and the results of data entry without looking at the underlying code.
  • Usability Testing: A combination of testing the user experience with the overall functionality of the website. A method is devised that will test all aspects of the site (such as content and navigation), and the application is improved after analysis of the results.
  • Interface Testing: Checks all communication processes between the application server and web server, as well as testing that all error messages are displayed correctly.
  • Compatibility Testing: Checks that the website functions correctly across different browsers (e.g. Google Chrome, Microsoft Edge), devices (e.g. desktop, smartphone, tablet) and operating systems (e.g. Windows, iOS).
  • Performance Testing: Tests that the application performs well and responds promptly to user input. This can involve testing the application to see how it behaves at varying internet connection speeds, simulating normal and peak loads, and stress testing to determine the how much load the application can sustain before breaking.
  • Security Testing: Checks to ensure the application is secure against malicious actions, for example by testing whether pages that should be secure can be accessed without authorisation or that open sessions are closed after a user leaves the site (or after a period of inactivity).
What are website debugging methods?

The first step to debugging a running website is to open the console (usually accessed by pressing the F12 key in a browser), granting access to information on the site’s performance and allowing the developer to see the underlying HTML code and monitor any error messages. The developer console can also be used to simulate a different device format, such as testing a smartphone screen layout within a desktop browser.

Third party website debugging tools such as Google’s Search Console are also available that can provide extensive status reports and automatically identify any errors.
What are website coding techniques?
The basic structure of a web page is written in HTML, typically with a separate HTML file for each page of the site (though pages can be automatically generated from templates for more complex websites). Styling for the website (e.g. colours, margins, padding) should be abstracted out to a single separate CSS file which can be referenced in the HTML files. More complex site functionality, such as automatically-resizing components, can be accomplished using plugins such as Bootstrap. PHP code can be embedded into HTML documents to execute SQL queries to a connected MySQL database.
